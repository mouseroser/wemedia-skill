# 国产大模型开发者视角——我用哪个？为什么？（M级）

## 核心观点

**我的主力：MiniMax-M2.7**

用它做什么：
- 复杂任务推理和规划
- 代码生成和 review
- 主 agent 编排

为什么选它：
- 推理能力强，复杂任务表现稳定
- 作为主力机，日常开发够用
- Function Calling 可用

**我放弃/不推荐的：**
- 百炼：工具链复杂，学习成本高，我的场景用不上阿里生态
- 文心4.0：中文强但 API 体验一般，Agent 场景不成熟
- 智谱GLM：开源生态好，但我没有私有部署需求

**我观察但不主力用的：**
- Kimi：长文本场景强，但我主要做代码和推理，不是它的目标用户
- 智谱：私有部署需求没有

---

## 差异化叙述方式
用"我为什么放弃XX"比"我为什么选XX"更有说服力
- 避免写成参数对比表
- 用实际场景：我用它做了什么，结果是什么

---

## 爆款因素
1. 有立场：读者喜欢"真实评价"而不是"云评测"
2. 有细节：具体失败案例让人印象深刻
3. 可讨论：说百炼/文心不好会引发反驳，互动率高
