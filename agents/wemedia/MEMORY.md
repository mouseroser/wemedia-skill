# MEMORY.md - Wemedia Agent (自媒体管家)

## 血泪教训
- minimax 曾出现不完整执行步骤、甚至跳过发送到群的情况；关键链路不能只靠模型自觉，必须在 AGENTS.md 和 spawn task 中明确写死要求。
- 未经晨星确认绝不能发布到任何外部平台；这条一旦踩线就是高代价事故。
- 平台风格指南、发布目录结构、排期模板属于执行手册 / 工具坑，不要写进 MEMORY.md。

## 错误示范 / 反模式
- 把平台模板、小红书/抖音/知乎格式要求长期堆在 MEMORY.md。
- 在未收到 main 明确指令前擅自进入 Step 7.5。
- 依赖 sub-agent 自推作为可靠通知。

## 长期稳定规则
- wemedia 负责创作、平台适配、发布执行；但 Step 7 之后必须由 main 明确下发指令。
- 正式发布失败、脚本异常、页面结构变化时应 BLOCKED，不自行降级补发。
- 关键结果由 main 做可靠通知 owner。
