## 执行摘要

本方案设计一套基于 OpenClaw 平台的小红书 **AI 辅助运营系统**，通过多 Agent 协作实现内容策划、创作辅助、数据分析、评论管理等功能。方案严格遵循"合规第一、人工执行门控"的宪法约束，定位为**辅助工具**而非自动化机器人。

核心价值主张：
- 将运营效率提升 200-300%（主要在选题、内容生成、数据分析环节）
- 保持内容质量（AI 生成 → AI 审核 → 人工终审三阶段）
- 零封号风险（所有平台操作由人工执行）
- 月运营成本控制在 ¥300-800 以内

## 一、Agent 架构设计

### 1.1 Agent 角色定义

| Agent | 角色名 | 职责 | 对应 OpenClaw agentId |
|-------|--------|------|---------------------|
| 选题策划师 | 内容雷达 | 热点追踪、竞品分析、选题推荐 | `xhs-radar` |
| 内容创作师 | 笔记工匠 | 文案生成、标题优化、标签推荐 | `xhs-writer` |
| 视觉设计师 | 封面大师 | 封面设计建议、配图方案、视觉风格 | `xhs-visual` |
| 数据分析师 | 数据侦探 | 运营数据追踪、效果分析、策略优化 | `xhs-analyst` |
| 评论管家 | 互动达人 | 评论分析、回复建议、用户洞察 | `xhs-social` |
| 运营总监 | 调度中心 | 跨 Agent 协调、任务调度、质量把关 | `xhs-director` |

### 1.2 协作流程

```
[每日运营周期]

06:00 ── xhs-radar 启动 ──┐
  │  热点扫描                │
  │  竞品更新                │
  │  选题推荐（3-5 个）      │
  └──── 产出: 选题清单 ──────┘
               │
08:00 ── 人工选择选题 ──────────── [人工门控 1]
               │
09:00 ── xhs-writer 接力 ───┐
  │  内容生成（基于选题）     │
  │  标题 A/B 方案           │
  │  标签推荐                │
  │  SEO 关键词布局          │
  └──── 产出: 内容草稿 ──────┘
               │
         xhs-visual 并行 ───┐
  │  封面设计方案             │
  │  配图建议                │
  └──── 产出: 视觉方案 ──────┘
               │
12:00 ── 人工审核 + 修改 ───────── [人工门控 2]
               │
12:30-14:00 ── 人工发布 ───────── [人工执行]
               │
15:00 ── xhs-social 监控 ───┐
  │  评论分析                │
  │  回复建议生成            │
  └──── 产出: 回复建议 ──────┘
               │
         人工确认 + 回复 ────────── [人工门控 3]
               │
22:00 ── xhs-analyst 日报 ──┐
  │  当日数据汇总            │
  │  内容表现分析            │
  │  策略调整建议            │
  └──── 产出: 日报 ──────────┘
               │
    xhs-director 周报 ──────┐
  │  周度数据趋势            │
  │  内容策略复盘            │
  │  下周选题方向            │
  └──── 产出: 周报 ──────────┘
```

### 1.3 Agent 间信息传递

```
                 xhs-director (调度中心)
                /     |      |       \
         xhs-radar  xhs-writer  xhs-visual  xhs-analyst
                        |
                    xhs-social

信息流：
- xhs-radar → xhs-writer: 选题清单 + 关键词建议
- xhs-writer → xhs-visual: 内容主题 + 风格要求
- xhs-social → xhs-analyst: 互动数据 + 用户反馈
- xhs-analyst → xhs-radar: 数据洞察 → 影响下一轮选题
- xhs-director: 读取所有 Agent 产出，协调优先级
```

**文件传递机制**（基于 OpenClaw 单写者原则）：
- 每个 Agent 产出写入 `~/.openclaw/workspace/agents/xhs-{role}/` 目录
- 共享上下文写入 `~/.openclaw/workspace/shared-context/xhs/`
- Agent 间通过文件系统传递，不依赖消息队列

## 二、功能模块详细设计

### 模块 1: 内容雷达（xhs-radar）

**功能**：
1. **热点追踪**：监控小红书热搜榜、热门话题标签、行业关键词趋势
2. **竞品监控**：追踪 5-10 个对标竞品账号的内容策略、发布频率、互动数据
3. **选题推荐**：基于热点 + 竞品 + 历史表现，生成每日 3-5 个选题建议

**数据来源**：
- 小红书 Web 端公开页面（通过浏览器工具抓取，不使用非官方 API）
- 第三方数据平台 API（千瓜数据、新榜小红书版）
- RSS 订阅行业资讯

**技术实现**：
```yaml
# OpenClaw Agent 配置示例
agent:
  id: xhs-radar
  model: gemini/gemini-3.1-pro-preview  # 适合信息检索和摘要
  cron: "0 6 * * *"  # 每天 6:00 执行
  skills:
    - web_fetch  # 抓取公开页面
    - tavily     # 搜索引擎
  output: ~/.openclaw/workspace/agents/xhs-radar/daily-topics-{date}.md
```

**输出格式**：
```markdown
# 每日选题推荐 | 2026-03-16

## 🔥 热点选题（时效性强）
1. [选题标题] — 热度: ⭐⭐⭐⭐⭐ | 竞争度: ⭐⭐⭐
   - 切入角度建议: ...
   - 参考笔记: ...
   - 关键词: ...

## 📈 趋势选题（长尾价值）
1. ...

## 🎯 竞品灵感
1. [竞品账号] 近期爆款: [笔记标题] (互动量 xxx)
   - 可借鉴点: ...
   - 差异化方向: ...
```

### 模块 2: 笔记工匠（xhs-writer）

**功能**：
1. **内容生成**：基于选题生成完整笔记草稿（标题、正文、标签）
2. **标题优化**：每个选题生成 3-5 个标题变体（A/B 测试用）
3. **SEO 优化**：关键词密度检查、搜索意图匹配
4. **风格适配**：根据账号人设调整语气、用词

**内容生成流程**：
```
输入: 选题 + 关键词 + 账号人设 + 历史爆款风格
     ↓
[xhs-writer Agent]
     ↓
产出: 
  ├── 标题方案 (3-5 个) + 推荐理由
  ├── 正文草稿 (800-1500 字)
  ├── 标签推荐 (8-15 个) + 搜索量估算
  ├── 发布时间建议
  └── AIGC 标注提醒
```

**质量控制**：
- 原创度检测：与互联网内容对比，重复率 < 20%
- 可读性评分：段落长度、emoji 使用、口语化程度
- 敏感词过滤：平台违禁词库 + 行业敏感词
- 宪法合规检查：自动检查是否违反 C1-C6 约束

**技术实现**：
```yaml
agent:
  id: xhs-writer
  model: anthropic/claude-opus-4-6  # 适合高质量中文内容创作
  thinking: medium
  skills:
    - web_fetch  # 参考竞品内容
  context:
    - shared-context/xhs/account-persona.md  # 账号人设
    - shared-context/xhs/content-style-guide.md  # 内容风格指南
    - shared-context/xhs/banned-words.md  # 违禁词库
```

### 模块 3: 封面大师（xhs-visual）

**功能**：
1. **封面设计方案**：基于内容主题，生成封面设计描述（构图、配色、文字）
2. **配图建议**：推荐图片风格、拍摄角度、后期处理
3. **Prompt 工程**：为 AI 图片生成工具（Midjourney/DALL-E）生成精准 prompt

**注意**：
- 封面图片的最终制作由人工完成或使用 AI 工具后人工审核
- Agent 不直接生成最终图片（避免 AI 痕迹过重）
- 推荐真实照片 + 文字排版的混合风格

---

### 模块 4: 数据侦探（xhs-analyst）

**功能**：
1. **日度数据追踪**：
   - 笔记曝光量、互动率（点赞/收藏/评论/转发）
   - 粉丝增减趋势
   - 搜索关键词排名
2. **内容分析**：
   - 哪类内容互动率最高
   - 最佳发布时间分析
   - 标题关键词效果
3. **策略建议**：
   - 数据驱动的内容方向调整
   - 粉丝画像分析（性别、年龄、地域、兴趣）
   - 增长瓶颈诊断

**数据采集方式**：
- 小红书创作者后台数据（需要人工截图或手动录入）
- 第三方数据平台 API（如千瓜数据 API）
- Agent 可引导人工在特定页面截图，然后用 image 工具分析

**数据存储**：
```
~/.openclaw/workspace/agents/xhs-analyst/
  ├── daily/
  │   ├── 2026-03-16.md
  │   └── ...
  ├── weekly/
  │   ├── week-2026-12.md
  │   └── ...
  └── insights/
      ├── content-performance.md  # 内容表现长期追踪
      └── audience-insights.md    # 粉丝洞察
```

---

### 模块 5: 互动达人（xhs-social）

**功能**：
1. **评论分析**：新评论通知、情感分析、关键问题识别
2. **回复建议**：基于评论内容和账号人设，生成回复建议
3. **用户洞察**：高频互动用户识别、潜在高价值粉丝标记
4. **舆情预警**：负面评论/争议内容及时提醒

**工作流程**：
```
评论数据输入（人工提供截图或文本）
     ↓
[xhs-social Agent 分析]
     ↓
产出:
  ├── 评论分类（问题/赞美/建议/负面/推广）
  ├── 优先回复列表（按重要性排序）
  ├── 每条评论的回复建议（2-3 个版本）
  └── 舆情预警（如有负面趋势）
     ↓
[人工确认 + 手动回复]  ← 人工门控
```

---

### 模块 6: 调度中心（xhs-director）

**功能**：
1. **任务编排**：按日/周/月周期驱动各 Agent 工作
2. **质量把关**：在内容交付人工之前做最终质量检查
3. **优先级管理**：热点时效性内容优先、常规内容排期
4. **资源协调**：API 调用预算控制、Agent 负载均衡

**实现方式**：利用 OpenClaw 的 cron 和 heartbeat 机制

```yaml
# 调度计划
cron:
  daily-radar: "0 6 * * *"      # 06:00 触发选题扫描
  daily-report: "0 22 * * *"    # 22:00 触发日报
  weekly-review: "0 10 * * 1"   # 周一 10:00 周报
  social-check: "0 */3 * * *"   # 每 3 小时检查评论
```

## 三、实施路线图

### Phase 1: 基础搭建（第 1-4 周）

**目标**：搭建最小可用系统，验证核心流程

| 周次 | 任务 | 交付物 |
|------|------|--------|
| W1 | 账号注册 + 定位 + 人设设计 | 账号资料文档 |
| W1 | OpenClaw Agent 配置（xhs-radar + xhs-writer） | Agent 配置文件 |
| W2 | 内容风格指南制定 | content-style-guide.md |
| W2 | 违禁词库搭建 | banned-words.md |

**KPI**：
- 完成 5 篇内容发布
- 内容准备时间从 2 小时/篇 → 1 小时/篇
- 零违规记录

### Phase 2: 能力扩展（第 5-8 周）

**目标**：启用数据分析和评论管理模块

| 周次 | 任务 | 交付物 |
|------|------|--------|
| W5 | 部署 xhs-analyst + xhs-social | Agent 配置文件 |
| W5 | 数据采集流程建立 | 数据模板 |

| W8 | Phase 2 复盘 + Phase 3 规划 | 复盘报告 |

**KPI**：
- 内容互动率 > 行业平均（3-5%）
- 评论回复率 > 80%
- 粉丝增长 200+/月

### Phase 3: 优化迭代（第 9-12 周）

**目标**：基于数据优化策略，形成稳定运营节奏

| 周次 | 任务 | 交付物 |

**KPI**：
- 内容准备时间 < 30 分钟/篇
- 月度粉丝增长 500+
- 单篇平均互动量 > 50
- 月运营成本 < ¥800

## 四、成本估算

### 月度运营成本

| 项目 | 估算费用/月 | 说明 |
|------|-----------|------|
| AI 模型调用（选题+内容+分析） | ¥200-400 | 基于 20 篇/月工作量 |
| 第三方数据平台 | ¥0-200 | 千瓜数据基础版 / 免费方案 |
| OpenClaw 平台费用 | ¥0 | 自托管 |
| 人工时间成本 | ¥0（自己运营）| 预计每天 1-2 小时 |
| **合计** | **¥200-600/月** | |

### 成本优化策略
- 使用 Gemini 做信息检索（成本低）
- 使用 Claude 做高质量内容创作（必要投入）
- 数据分析使用轻量模型（如 GPT-4o-mini）
- 按需调用，非全天候运行

## 六、风险评估与应对

| 风险 | 等级 | 应对策略 |
|------|------|----------|
| 平台规则变更导致策略失效 | 🟡 中 | xhs-radar 监控规则变更；模块化设计支持快速调整 |
| AI 生成内容被识别为低质量 | 🟡 中 | 人工终审把关；持续优化 prompt；混合人工创作 |
| 运营效果不及预期 | 🟡 中 | 4 周为一个评估周期；数据驱动策略调整；及时止损 |
| 竞品抓取被反爬 | 🟢 低 | 降低频率；切换数据源；使用第三方平台数据 |
| 账号被误判限流 | 🟢 低 | 申诉渠道准备；内容合规自查；发布频率控制 |

**注意**：由于方案已排除自动化发帖和刷量行为，高风险项（账号封禁、法律合规）已被宪法约束从根源上消除。
