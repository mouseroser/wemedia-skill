## 执行摘要

本方案设计了一套基于 OpenClaw 平台的**小红书 AI 辅助运营系统**，通过 4 个 AI Agent 协作，覆盖内容策划、内容创作、数据分析、互动管理四大核心环节。

### 核心定位
**AI 辅助 + 受控数据读取 + 人工写入执行** — AI Agent 负责"想"和"准备"，人工负责"执行"和"判断"。

### 关键结论
1. **推荐路线**: AI 辅助 + 人工执行模式（非自动化运营）
2. **Agent 架构**: 4 个专职 Agent（雷达 + 工匠 + 互动 + 分析）
3. **MCP 集成**: 采用**方案 C+**（增强型只读集成）— 仅使用 xiaohongshu-mcp 的数据读取功能，禁止写入自动化
4. **实施周期**: 3 阶段 12 周渐进式启用
5. **月运营成本**: ¥300-1000（根据模型选择）
6. **核心价值**: 内容准备效率提升显著，封号风险极低

## 二、xiaohongshu-mcp 集成方案 C+

### 2.1 方案定位

**方案 C+（增强型只读集成）**：
- 在方案 C（纯只读）基础上，将 MCP 只读能力深度嵌入现有 Agent 工作流
- 写入操作全部禁止，保持人工执行
- 不开发独立 Gateway，使用 MCPorter 配置实现访问控制

### 2.2 功能分级矩阵

| 功能 | MCP 工具名 | 灯号 | 执行方式 | 风险等级 | 日均上限 | 回退方案 |
|------|-----------|------|---------|---------|---------|---------|
| 登录状态检查 | `check_login_status` | 🟢 绿灯 | MCP 自动 | 低 | 无限制 | 人工检查 |
| 获取登录二维码 | `get_login_qrcode` | 🟢 绿灯 | MCP + 人工扫码 | 低 | 按需 | 人工登录 |
| 删除 Cookies | `delete_cookies` | 🟢 绿灯 | MCP 自动 | 低 | 按需 | 手动清除 |
| 搜索内容 | `search_feeds` | 🟢 绿灯 | MCP 自动 | 中 | ≤20 | 人工搜索 + 截图 |
| 首页推荐 | `list_feeds` | 🟢 绿灯 | MCP 自动 | 低 | ≤10 | 人工浏览 |
| 帖子详情 | `get_feed_detail` | 🟢 绿灯 | MCP 自动 | 中 | ≤40 | 人工查看 + 截图 |
| 用户主页 | `user_profile` | 🟡 黄灯 | MCP 自动 | 中高 | ≤10 | 人工查看 |
| **发布图文** | `publish_content` | 🔴 红灯 | **禁止** | 极高 | — | 人工发布 |
| **发布视频** | `publish_with_video` | 🔴 红灯 | **禁止** | 极高 | — | 人工发布 |
| **发表评论** | `post_comment_to_feed` | 🔴 红灯 | **禁止** | 极高 | — | AI 建议 + 人工回复 |
| **回复评论** | `reply_comment_in_feed` | 🔴 红灯 | **禁止** | 极高 | — | AI 建议 + 人工回复 |
| **点赞** | `like_feed` | 🔴 红灯 | **禁止** | 高 | — | 人工点赞 |
| **收藏** | `favorite_feed` | 🔴 红灯 | **禁止** | 高 | — | 人工收藏 |

**日均调用总量**: 约 23-40 次，远低于宪法约束的 200 次/天上限，安全边际充足。

### 2.3 MCP Gateway 实现路径（v2.0 明确）

**核心原则**: 使用 MCPorter 配置实现访问控制，而非独立开发中间件。

#### 实现方案

| 层级 | 实现方式 | 说明 |
|------|---------|------|
| 白名单 | MCPorter server 注册 | 仅注册允许使用的工具 |
| 黑名单 | 硬编码拒绝 | 写入类工具不注册 |
| 频率限制 | OpenClaw 配置 | 全局 ≤200/天，单工具限流 |
| 审计日志 | 文件系统日志 | 记录每次调用 |
| 熔断 | wrapper 脚本 | 连续失败 3 次自动暂停 |

**预计开发量**: 2-4 小时（MVP 方式）

## 三、已知失败信号清单（v2.0 新增）

### 3.1 风险等级分类

| 风险类型 | 描述 | 触发条件 | 严重程度 |
|---------|------|---------|---------|
| 账号封禁 | 使用非官方自动化工具被检测后封号 | 高频调用、异常行为模式 | 致命 |
| 限流处罚 | 笔记被平台减少推荐流量 | 自动化操作触发风控 | 高 |
| 验证码触发 | 登录或操作时要求图形验证 | 异常行为模式、频繁请求 | 中 |
| Cookies 失效 | 登录状态过期需重新扫码 | 有效期届满、异地登录 | 中 |
| 实名认证触发 | 新号/未实名账号被要求认证 | 账号权重低、频繁操作 | 中 |
| IP 限制 | 请求被限制或封禁 | 频繁请求、反爬升级 | 中 |

### 3.2 降级策略

```
MCP 降级瀑布：
Level 0: MCP 正常 ──────────── 全功能使用
Level 1: MCP 限流 ──────────── 降低调用频率 50%
Level 2: MCP 熔断 ──────────── 停用 MCP，使用备用数据源
Level 3: Cookies 失效 ────────── 停用 MCP，等待人工重新登录
Level 4: 账号异常 ──────────── 停用一切自动化，仅人工操作

每个级别的回退方案：
┌─────────────┬──────────────────────────────────┐
│ Agent       │ 降级回退                           │
├─────────────┼──────────────────────────────────┤
│ xhs-radar   │ web_search + 第三方平台 + 人工搜索  │
│ xhs-social  │ 人工截图 + image 工具分析           │
│ xhs-analyst │ 人工导出创作者后台 CSV + AI 分析    │
│ xhs-writer  │ 无影响（不使用 MCP）               │
└─────────────┴──────────────────────────────────┘
```

## 五、宪法约束（v2.0 更新）

### C1: 合规第一
- ✅ 官方合规：创作者后台人工导出、人工浏览
- ✅ 非官方只读：xiaohongshu-mcp 低频只读集成（条件允许）
- ❌ 写入禁止：所有 MCP 写入类操作禁止进入生产

### C2: 质量优先于数量
- 三阶段审核（选题 → 内容 → 发布）
- AI 质量评分 > 7/10 方可进入下一阶段

### C3: 人工执行门控
- 选题确认：人工门控 1
- 内容审核：人工门控 2
- 评论回复：人工门控 3
- 发布执行：人工在官方客户端完成

### C4: 数据安全与隐私
- 不采集个人信息
- 日志脱敏处理
- Cookies 文件权限 600

### C5: 透明度
- 完整审计日志
- 监控告警机制
- 定期合规复核

### C6: 可逆性
- MCP 可一键停用
- 回退方案完整
- 人工流程随时启用
