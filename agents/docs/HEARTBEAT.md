# HEARTBEAT.md - Docs Agent 健康检查

## 健康检查（每次心跳时运行）

### 文档完整性
检查关键文档是否存在且最新：
```bash
# 检查文档目录
ls -la docs/ 2>/dev/null || echo "文档目录需要检查"

# 检查 README
ls -la README.md 2>/dev/null
```

### 待撰写任务
检查是否有未完成的文档任务：
```bash
# 读取最近的 memory 日志
tail -n 50 memory/$(date +%Y-%m-%d).md
```

### 文档同步检查
检查代码变更是否需要更新文档：
```bash
# 检查最近的 git 提交
git log --oneline -10
```

### 故障恢复
如果发现中断的文档任务：
1. 读取 `memory/YYYY-MM-DD.md` 确认上下文
2. 检查是哪个文档中断（技术文档/用户指南/API 文档）
3. 推送状态到项目文档群（-5095976145）+ 监控群（-5131273722）
4. 等待 main 指令

---

**原则**:
- 不要自己决定是否继续中断的文档
- 文档中的示例代码必须验证能跑
- 代码变更后主动检查是否需要更新文档
