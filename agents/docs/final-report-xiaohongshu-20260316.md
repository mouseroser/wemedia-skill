## 执行摘要

本方案设计了一套基于 OpenClaw 平台的**小红书 AI 辅助运营系统**，通过 4 个 AI Agent 协作，覆盖内容策划、内容创作、数据分析、互动管理四大核心环节。

### 核心定位
**AI 辅助运营系统** — AI Agent 负责"想"和"准备"，人工负责"执行"和"判断"。

### 关键结论
1. **推荐路线**: AI 辅助 + 人工执行模式（非自动化运营）
2. **Agent 架构**: 4 个专职 Agent（雷达 + 工匠 + 互动 + 分析）
3. **实施周期**: 3 阶段 12 周渐进式启用
4. **月运营成本**: ¥300-1000（根据模型选择）
5. **核心价值**: 内容准备效率提升 200-300%，封号风险极低

### 为何不做全自动化
原始任务提到"自动发帖、点赞、回复、引流"，经星鉴流水线评估后明确排除：
- **自动发帖**: 小红书无官方发布 API，非官方方案封号率 >90%（7-30天内）
- **自动点赞/收藏**: 构成流量造假，违反《反不正当竞争法》
- **自动评论**: 垃圾信息，损害社区生态，触发平台反制
- **自动引流**: 小红书对站外引流检测严格，违规引流是封号高发场景

以上行为的风险远超收益。方案转而聚焦于 AI 在"策划、创作、分析"三个环节的辅助价值。

## 一、Agent 架构设计

### 1.1 精简后 4 Agent 体系

| Agent ID | 角色名 | 核心职责 | 推荐模型 |
|----------|--------|----------|----------|
| `xhs-radar` | 内容雷达 | 热点追踪、竞品分析、选题推荐 | gemini-3.1-pro-preview |
| `xhs-writer` | 内容工匠 | 文案生成、标题优化、视觉方案、质量自检 | claude-opus-4-6 |
| `xhs-social` | 互动管家 | 评论分析、回复建议、用户洞察、舆情预警 | gpt-5.4 |
| `xhs-analyst` | 数据分析 | 运营数据追踪、效果分析、策略建议 | gemini-3.1-pro-preview |

**说明**:
- 原 6 Agent 方案经复核后精简：xhs-visual 合并入 xhs-writer，xhs-director 由 main agent cron/heartbeat 承接
- 减少 2 个 Agent 降低系统复杂度和维护成本
- xhs-radar 和 xhs-analyst 使用 Gemini（信息检索成本低），xhs-writer 使用 Claude（中文创作质量高）

## 二、功能模块详细设计

### 模块 1: 内容雷达 (xhs-radar)

| 功能 | 说明 |
|------|------|
| 热点追踪 | 监控小红书热搜、热门话题、行业关键词 |
| 竞品监控 | 追踪 5-10 个对标账号的内容和数据 |
| 选题推荐 | 每日 3-5 个选题 + 切入角度 + 关键词 |

**数据来源优先级**:
1. 小红书创作者后台（人工导出 CSV）
2. 第三方平台 API（千瓜数据/新榜）
3. 浏览器工具抓取公开页面（补充）

**触发**: cron `0 6 * * *`（每天 06:00）

**输出模板**:
```markdown
# 每日选题 | {date}

## 🔥 热点选题 (时效 24-48h)
1. [标题] | 热度⭐⭐⭐⭐ 竞争⭐⭐
   角度: ... | 关键词: ... | 参考: ...

## 📈 趋势选题 (长尾价值)
1. ...

## 🎯 竞品启发
1. [账号] 爆款 [标题] (互动 xxx)
   可借鉴: ... | 差异化: ...
```

### 模块 2: 内容工匠 (xhs-writer)

| 功能 | 说明 |
|------|------|
| 内容生成 | 基于选题生成完整笔记草稿 |
| 标题优化 | 每篇 3-5 个标题变体 |
| 标签推荐 | 8-15 个标签 + 搜索量估算 |
| 封面方案 | 封面构图、配色、文字建议 + AI 图片 prompt |
| SEO 优化 | 关键词密度、搜索意图匹配 |
| 质量自检 | 原创度、可读性、敏感词、宪法合规 |

**质量控制标准**:
| 检查项 | 合格线 |
|--------|--------|
| 原创度 | 与互联网内容重复率 < 20% |
| 完整性 | 标题 + 封面方案 + 正文 + 标签 |
| 可读性 | AI 评分 > 7/10 |
| 合规性 | 零敏感词、零违禁品、零虚假宣传 |
| AIGC 标注 | 草稿中自动包含标注提醒 |

**触发**: 人工确认选题后手动触发

### 模块 3: 互动管家 (xhs-social)

| 功能 | 说明 |
|------|------|
| 评论分析 | 新评论分类（问题/赞美/建议/负面/推广） |
| 回复建议 | 每条评论 2-3 个回复版本 |
| 用户洞察 | 高频互动用户识别 |
| 舆情预警 | 负面趋势及时提醒 |

**工作流程**:
1. 人工提供评论数据（截图/复制文本）
2. Agent 分析 + 生成回复建议
3. 人工确认 + 手动回复

**触发**: cron `0 */3 * * *`（每 3 小时提醒检查评论）

### 模块 4: 数据分析 (xhs-analyst)

| 功能 | 说明 |
|------|------|
| 日报 | 当日发布内容表现、互动数据 |
| 周报 | 周度趋势、内容排行、策略复盘 |
| 洞察 | 内容类型效果对比、最佳发布时间 |

**数据输入方式**:
- 人工截图小红书后台 → Agent 用 image 工具提取数据
- 人工录入到标准化模板 → Agent 分析
- 第三方 API 直接拉取

**触发**: cron `0 22 * * *`（每天 22:00）

## 五、冷启动策略

### 第 1 周: 养号期
- 每天浏览 30-60 分钟，点赞、收藏同赛道优质笔记
- 在 5-10 篇笔记下发表真实评论（非引流）
- 完善个人资料（头像、简介、背景图）
- **不发布任何内容**

### 第 2 周: 测试期
- 发布 2 篇测试笔记（纯人工创作，不用 AI）
- 观察初始流量和互动数据
- 分析小红书给予的初始推荐量级

### 第 3-4 周: 正式启动
- 启用 xhs-radar + xhs-writer Agent
- 发布 5-8 篇内容（AI 辅助 + 人工审核发布）
- 根据数据反馈调整内容方向

### 关键认知
前 10 篇笔记的质量和互动数据，直接决定小红书算法给予账号的初始权重。宁可少发高质量笔记，不要多发低质量内容。
