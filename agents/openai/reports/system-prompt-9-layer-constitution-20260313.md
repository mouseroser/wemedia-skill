# OpenAI 宪法简报 - OpenClaw System Prompt 9层架构评估

## 1. 范围（Scope）
- **包含内容**：评估 OpenClaw System Prompt 的 L1 到 L9 完整架构，包括静态注入（如 L7 Workspace Files）、动态注入（L8 Bootstrap Hook）及上下文叠加机制。重点聚焦于上下文膨胀、冲突解决机制、执行优先级（特别是 L7 与 L8 之间的覆盖关系）及 Token 容量管理（如 `bootstrapMaxChars` 机制）。
- **不包含内容**：不涉及底层的具体模型训练细节、模型架构本身的评测，以及非 OpenClaw 框架之外的其他智能体框架的横向对比；不涉及底层大语言模型的微调机制。

## 2. 定义（Definitions）
- **L1-L6 (Core Framework)**：由 OpenClaw 框架底层生成的、用户不可直接编辑的系统提示词部分（如身份、安全沙箱、工具注册、模型别名等）。
- **L7 (Workspace Files)**：用户可控的静态配置文件，如 `SOUL.md`、`AGENTS.md`、`MEMORY.md` 等，随用例增多易引发上下文膨胀。
- **L8 (Bootstrap Hook)**：允许用户通过脚本动态注入额外上下文的机制，如 `extra-files`、`before_prompt_build` 等，可定制性强但可能引入冲突或性能开销。
- **L9 (Inbound Context)**：实时流量或当次请求的特定上下文、历史对话及当前文件流转状态。
- **上下文膨胀 (Context Bloat)**：注入系统提示词的文件和信息过多，导致逼近甚至超出大模型上下文窗口限制（或 `bootstrapMaxChars` 阈值）。
- **指令冲突 (Instruction Conflict)**：不同层级（如 L7 静态文件和 L8 动态脚本）对同一操作提供矛盾的规则和指示。

## 3. 判定标准（Criteria）
- **可行性标准**：
  - **覆盖与优先级清晰度**：能否明确阐述当 L7 和 L8 发生冲突时，系统的判定优先级和覆盖机制。
  - **性能与截断机制的健壮性**：评估系统面对上下文膨胀触发 `bootstrapMaxChars` 时，截断策略的合理性（是智能截断、丢弃还是报错），以及 L8 动态脚本执行对 Prompt 构建的实际延迟。
  - **安全与越权防控**：L8 的脚本权限边界是否明确，能否防御针对 L1-L6 的越权篡改和外部恶意注入。
- **质量标准**：
  - 提出的优化建议或评估结论必须基于实际的框架工作流（即 9层叠加的顺序与依赖）。
  - 需要解答 Gemini 快速扫描中提出的 4 个"待验证假设"。

## 4. 禁止项（Prohibitions）
- **禁止纯理论推演**：不允许在没有具体场景和验证支撑的情况下，凭空推断 L8 的权限可以随意覆盖 L1-L6。
- **禁止忽视黑盒层**：评估时不得略过 L1-L6 的基础作用，必须承认这些基础层对整体安全和功能的前提约束。
- **禁止片面关注应用层**：不得只谈论 L7 的 Prompt 编写技巧（如如何写好 SOUL.md），而忽略 L8 动态机制及整体架构层级间的协同和冲突。
- **禁止未加证明的安全假设**：不允许假定系统默认免疫任何通过 L8 或 L9 进行的 Prompt Injection 攻击。

## 5. 证据门槛（Evidence Threshold）
- **优先级验证证据**：必须提供明确的逻辑推导或（如可能）文档引用，证明 L8 在构建顺序上与 L7 的相互作用及覆盖关系（回应假设1）。
- **权限边界证据**：必须明确指出 `before_prompt_build` 等钩子在 OpenClaw 实际设计中是否具备修改/拦截 L1-L6 的底层权限（回应假设2）。
- **截断策略证据**：需要给出系统处理 Token 溢出的具体行为描述（智能截断 vs 随机丢弃），说明对 L8/L9 影响的程度（回应假设3）。
- **动态模型切换证据**：必须分析 L4（Model Aliases）在多轮对话中动态切换时，对 L5-L9 状态维持的实际影响（回应假设4）。

## 6. 输出格式（Output Format）
最终的深度研究和评估报告必须包含以下结构：
1. **执行摘要 (Executive Summary)**：核心发现和架构风险概览。
2. **架构层级分析 (Layer-by-Layer Analysis)**：重点拆解 L1-L6 黑盒层、L7 静态层和 L8 动态层的交互。
3. **关键问题回应 (Key Issues Resolved)**：
   - 冲突与优先级机制解析
   - 上下文容量与截断策略评估
   - L8 权限边界与安全风险分析
4. **假设验证结论 (Hypothesis Verification)**：逐一回应 Gemini 扫描中的 4 个假设。
5. **优化与加固建议 (Recommendations)**：针对上下文膨胀和动态注入提出的具体改进方案。
