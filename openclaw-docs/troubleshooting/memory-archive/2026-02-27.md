# 2026-02-27 工作日志

## 生图群问题排查与修复
- 问题：生图群只显示文字 "image" 不显示图片
- 根因1：生图群 session 的 model 被 override 成 gemini-3-pro-image-preview（不支持 tool use），只能输出 `![Image](url)` 文字
- 根因2：main agent 有该群的旧 session，导致 binding 到 img-proxy 不生效，消息一直路由到 main
- 修复：删除 main 的旧 group session、移除群 systemPrompt、img-proxy 模型改为 minimax/MiniMax-M2.5
- ⚠️ 踩坑：sessions.json 里的旧 session 会抢占 binding 路由，必须删除旧 session 后重启 gateway
- 私聊生图流程也有问题：nano-banana announce 返回后，直接把 `![Image](url)` 文字发到 Telegram，显示为 "Image"
- 修复：更新 MEMORY.md 约定，收到 announce 后必须提取 URL → 下载 → message 发送图片 → NO_REPLY

## 踩坑：nano-banana announce 显示 failed 但图片实际生成成功
- 现象：spawn nano-banana 后 announce 显示 "❌ Subagent nano-banana failed"，但返回内容中包含图片 URL
- 原因：gemini-3-pro-image-preview 生图时先返回一条空 assistant 消息（content=[]），然后再返回包含图片的消息。OpenClaw 看到第一条空响应就判定为 failed
- 影响：不影响实际使用，图片确实生成了，只是 announce 状态标记错误
- 处理：收到 nano-banana 的 announce 时，无论 ✅ 还是 ❌，都检查返回内容中是否有图片 URL，有就下载发送
