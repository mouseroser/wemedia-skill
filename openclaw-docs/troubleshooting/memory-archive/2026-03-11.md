## 13:05 Announce 不可靠的双保险机制

晨星同意的解决方案。

问题：Sub-agent announce 是 best-effort，不保证到达。子 agent 完成了，但 main 可能收不到 announce。

解决方案：双保险机制。主链路：等待 announce（大部分情况下会到达）。备用链路：超时后检查文件（只在 announce 失败时触发）。

具体实现：Spawn 后：(1) 记录预期输出路径和 spawn 时间；(2) 等待 announce；(3) 如果 10 分钟后还没收到 announce：检查输出文件是否存在，如果存在且时间戳新鲜（< 2 分钟）→ 继续推进，如果不存在 → 推送告警到监控群。

优势：平衡可靠性和效率，不占用主私聊会话（只在超时时检查一次），不需要持续轮询，符合"主私聊不承载长编排"约束。

容错性：Announce 正常到达 → 正常推进；Announce 丢失但任务完成 → 超时后检查文件继续推进。
