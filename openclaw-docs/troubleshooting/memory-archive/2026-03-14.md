## 15:55 Control UI 三条 issue 分工结论

- 之前“Issue #44619 已随升级修复”的判断对这台机器来说过早；实测移除本地 `2s idle chat.history polling` 热修复后，WebChat 空闲时仍会漏掉 Telegram 侧更新，因此 `#44619` 已重新打开。
- `#45513` 应只聚焦在 `context-notice` overlay / layout 挡住输入框的问题，不再混入 token 口径讨论。
- `#45572` 是当前更适合承接“context warning 计算口径 / 数据源错误”的主线程：
  - chat 顶部 notice 现在读取 `session.inputTokens / session.contextTokens`
  - 但本机 session store 已出现 `inputTokens` 明显偏离 `totalTokens` 的情况，说明该字段不稳定地代表“当前上下文占用”
  - 本地 override 已改为优先使用 `totalTokens`，仅在缺失时回退到 `inputTokens`
- `#45794` 继续承接 chat pane 空白/错位问题；它与 `#45572` 的 token 口径问题相关，但不是同一个 bug。
- 这条结论值得记录，因为它能避免后续继续把同一类 Control UI 问题报到错误的 issue 里；但只记录分工和稳定结论，不记录 comment 级操作细节。
