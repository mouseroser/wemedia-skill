## 核心设计原则

1. **文件系统即集成层** - 不需要消息队列或数据库
2. **单写者原则** - 每个共享文件只有一个写者，多个读者
3. **调度保证协作** - 依赖关系通过执行顺序保证
4. **记忆即文件** - 脑子里的东西会消失，文件不会
5. **渐进式进化** - 让系统随使用自然生长

## 教训和经验

1. **Sub-agent 推送不可靠**：depth-2 限制导致 message 工具失效，关键推送由 main 补发
2. **Telegram 配置复杂**：allowFrom 有多个层级（全局/channel/group），需要正确配置
3. **Privacy Mode 必须关闭**：requireMention: false 只是 OpenClaw 配置，Telegram 端需要单独设置
4. **通配符 "*" 支持**：groups 配置可以用通配符简化
5. **LLM timeout 需要重试**：第一次失败不代表任务不可行，重试通常能解决
6. **文章内容获取困难**：小红书网页版限制，需要通过 App 截图或复制内容

1. **架构理解需要多次迭代**：从错误的独立 workspace → 共享 workspace → agent 工作目录 → 简化嵌套
