## 记忆系统 LLM 技术链路完整解析 (14:09)

回答晨星"记忆系统的 LLM 是怎么工作的"：

**4 个 AI 环节**：
1. **Smart Extraction**: `openai/gpt-oss-120b` (云端) - 从对话提取记忆候选、去重决策、合并
2. **Embedding**: `nomic-embed-text` (本地 Ollama) - 文本 → 768维向量
3. **Rerank**: `bge-reranker-v2-m3` (本地 Python sidecar, 端口 8765) - 检索结果二次排序
4. **Layer 3 Fallback**: NotebookLM (云端) - L2 不够时的深度推理

**Smart Extraction 3 次 LLM 调用**：
- 提取：最多 5 个候选记忆（6 类：profile/preferences/entities/events/cases/patterns）
- 去重：向量相似 → LLM 判断 create/merge/skipsupersede/support/contradict
- 合并：如果决策是 merge，再调 LLM 合并新旧记忆

**噪音过滤**：纯 embedding 相似度，不用 LLM。内置噪音原型库，LLM 返回 0 条时自动学习为新噪音。
