## 踩坑记录

### ❌ Telegram 群白名单用了错误字段 `allowGroups`
- 错误：`channels.telegram.allowGroups: [...]`
- 正确：`channels.telegram.groups: { "<chatId>": { ... } }`
- OpenClaw 校验严格，未知字段直接报错无法启动

### ❌ 群消息默认需要 @mention 才响应
- 现象：日志显示 `reason: "no-mention"` 跳过消息
- 解决：每个群配置里加 `"requireMention": false`

### ❌ webhook 模式下 getUpdates 返回空
- Telegram bot 如果被 polling 消费了 updates，getUpdates 拿不到历史数据
- 解决：从 OpenClaw 的 `/tmp/openclaw/` 日志里 grep chat ID

### ❌ openclaw doctor --fix 会清除未知配置字段
- 如果配置了错误字段，doctor 会自动移除
- 注意：改完配置后先确认字段名正确再重启

### ✅ Telegram 群 bindings 的 peer.id 格式
- 使用 `"tg:-xxxxxxxxxx"` 格式（带 tg: 前缀）

### ✅ groups 配置同时充当白名单
- `channels.telegram.groups` 配置了哪些群，哪些群就在白名单里
- 不需要额外的 allowlist 字段
