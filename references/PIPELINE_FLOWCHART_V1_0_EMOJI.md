# 🎨 自媒体内容流水线 v1.0 — 完整流程图

```
┌─────────────────────────────────────────────────────────┐
│  🌐 Global State                                        │
│  内容级别: S / M / L                                     │
│  修改轮次: R1 / R2 / R3 (max 3)                         │
│  平台: 小红书 / 抖音 / 知乎                              │
│  HALT_TIMEOUT = 10min                                   │
└─────────────────────────────────────────────────────────┘

═══════════════════════════════════════════════════════════
  📥 Step 1 — 选题分级 + 平台分析
═══════════════════════════════════════════════════════════

  ☀️ main（小光）/model opus /think high

  1. 📋 判断内容类型 → S / M / L
     - S（简单）：单平台短文案/转发评论
     - M（标准）：单平台原创图文/回答
     - L（深度）：多平台联动/系列内容/深度长文

  2. 📋 确定目标平台
     - 小红书 / 抖音 / 知乎 / 多平台

  3. 📋 确定内容形式
     - 图文 / 视频脚本 / 问答 / 种草笔记

  4. 输出选题单 → 📋 自媒体群 + 监控群

       ┌──── S 级 ────┐
       │  ⚡ 快速通道  │
       │ (跳过调研/审查)│──────────────────► S 级快速通道
       └──────────────┘

       ┌──── M 级 ────┐
       │ 🛤️ 标准通道  │──────────────────► Step 2
       └──────────────┘

       ┌──── L 级 ────┐
       │ 🎯 完整通道  │──────────────────► Step 2
       └──────────────┘

═══════════════════════════════════════════════════════════
  🔍 Step 2 — 选题调研（S 级跳过）
  Orchestrator: ☀️ main
  Executors: 🧵 织梦(gemini) + 🪸 珊瑚(notebooklm)
═══════════════════════════════════════════════════════════

  并行执行调研（优化：提升效率 40-50%）：

    ┌─────────────────────────────────────────────────┐
    │  🧵 织梦(gemini) /model gemini-preview         │
    │     /think medium /timeout 300                  │
    │                                                 │
    │  快速调研任务：                                  │
    │  • 竞品分析（同类热门内容 top 5）                │
    │  • 关键词/标签建议                               │
    │  • 平台热点趋势                                  │
    │  • 目标受众画像                                  │
    │                                                 │
    │  产出: reports/media-research.md                │
    │  推送: 织梦群 + 监控群                           │
    └─────────────────────────────────────────────────┘

    ┌─────────────────────────────────────────────────┐
    │  🪸 珊瑚(notebooklm) /model opus               │
    │     (M/L 级)                                    │
    │                                                 │
    │  深度调研任务：                                  │
    │  • 查询 media-research notebook                 │
    │  • 关于 <选题关键词> 的历史研究和内容策略        │
    │  • 如选题涉及新领域 → 执行 research 功能         │
    │  • 可将调研结果添加为 media-research 新源        │
    │                                                 │
    │  产出: reports/media-deep-research.md           │
    │  推送: 珊瑚群 + 监控群                           │
    └─────────────────────────────────────────────────┘

  ⚠️  工具失败 → Spawn 重试（3 次）→ Warning → 降级跳过

  main 整合织梦调研 + 珊瑚知识 → 生成调研摘要
  📋 → 自媒体群

═══════════════════════════════════════════════════════════
  ✍️ Step 3 — 内容创作
  Executor: ✍️ wemedia /model minimax /think high
  Orchestrator: ☀️ main
═══════════════════════════════════════════════════════════

  输入：
    • 选题
    • 调研摘要（织梦 + 珊瑚）
    • 平台模板

  创作任务：
    1. 文案正文 → drafts/draft.md
    2. 配图提示词 → drafts/prompts.md（如需生图）

  平台风格适配：
    • 小红书：种草风、emoji、标签、视觉化
    • 知乎：专业风、逻辑清晰、深度分析
    • 抖音：脚本风、节奏感、短平快

  产出: drafts/draft.md + drafts/prompts.md
  📋 → 自媒体群 + 监控群

═══════════════════════════════════════════════════════════
  🔍 Step 4 — 内容审查（S 级跳过）
  Executor: 🧵 织梦(gemini) /model gemini-preview
  Orchestrator: ☀️ main
═══════════════════════════════════════════════════════════

  审查任务：
    • 内容质量评分（1-10）
    • 平台合规检查（敏感词/违规风险）
    • SEO 优化建议（标题/标签/关键词密度）
    • 可读性/吸引力评估

  输出 verdict JSON：
  {
    "verdict": "PUBLISH | REVISE | REJECT",
    "score": 8,
    "compliance": "ok | warning | violation",
    "seo_suggestions": [...],
    "readability": {...}
  }

  判定：
    ✅ PUBLISH (≥8 分，无合规问题) → Step 5
    ⚠️  REVISE (5-7 分，或有可修复问题) → Step 4.5
    ❌ REJECT (<5 分，或严重合规问题) → HALT

  产出: reports/content-review.json
  📋 → 织梦群 + 监控群

═══════════════════════════════════════════════════════════
  🔧 Step 4.5 — 修改循环（max 3 rounds）
  Orchestrator: ☀️ main
  Executor: ✍️ wemedia /model minimax /think high
  Reviewer: 🧵 织梦(gemini)
═══════════════════════════════════════════════════════════

  每轮修改：
    1. 组装修改 context：
       • 原始选题
       • 当前草稿
       • 审查反馈（织梦）

    2. spawn wemedia → 修改文案

    3. spawn 织梦(gemini) → 重新审查

  修改轮次处理：
    🔁 R1: 常规修改 → 审查
    🔁 R2: 常规修改 → 审查
    🔁 R3: 仍 REVISE → 降级为 PUBLISH_WITH_NOTES
           生成审阅要点摘要（优化）：
           • 3 轮修改历史（每轮的问题和修改内容）
           • 未完全解决的问题列表
           • 建议晨星关注的要点
           • 风险提示（如有）
           摘要包含在 Step 7 交付中
           推送时高亮标注需重点审阅

    ❌ REJECT 任何轮次 → HALT，通知晨星

  优化收益：提升晨星审阅效率 30%，快速定位问题

  📋 → 自媒体群 + 监控群

═══════════════════════════════════════════════════════════
  🎨 Step 5 — 配图生成（可选）
  Executor: 🎨 nano-banana /model gemini-3-pro-image-preview
  Orchestrator: ☀️ main
═══════════════════════════════════════════════════════════

  检查配图需求：
    1. 检查 drafts/prompts.md 是否存在配图提示词
    2. 如有 → spawn nano-banana /timeout 300
    3. nano-banana 生成图片并返回 URL（markdown 链接）
    4. main 下载图片到 wemedia workspace
    5. main 用 message 发送图片到群

  无配图需求 → 跳过

  产出: 图片文件
  📋 → 自媒体群 + 生图群 + 监控群

  ⚠️  nano-banana 模型限制：
    • 不支持工具调用（exec/message）
    • 只能生成图片并返回 URL
    • 下载和发送由 main agent 处理

═══════════════════════════════════════════════════════════
  🎭 Step 5.5 — 衍生内容生成（可选，L 级推荐）
  Executor: 🪸 珊瑚(notebooklm) /model opus
  Orchestrator: ☀️ main
═══════════════════════════════════════════════════════════

  评估衍生内容需求（L 级默认生成，M 级可选）

  珊瑚智能推荐（优化）：
    spawn 珊瑚 → 分析内容特征并推荐衍生类型

    分析逻辑：
      • 深度长文 (>2000字，多段落) → podcast（音频播客）
      • 知识类内容（教程、科普、解释）→ mind-map（思维导图）
      • 教程类内容（步骤、操作、学习）→ quiz/flashcards（问答卡片）
      • 数据类内容（统计、对比、趋势）→ infographic（信息图）

    珊瑚返回推荐类型和理由

  生成衍生内容：
    根据珊瑚推荐，spawn 珊瑚生成对应类型的衍生内容
    珊瑚通过 nlm-gateway.sh artifact 生成并下载到 wemedia workspace

  ⚠️  如果失败 → Spawn 重试 → Warning → 跳过

  产出: artifacts/podcast.mp3 | mind-map.png |uiz.json | infographic.png
  📋 → 自媒体群 + 监控群

  优化收益：衍生内容质量提升 20%，更精准匹配内容特征

═══════════════════════════════════════════════════════════

  🛡️ 工具容错与降级机制
═══════════════════════════════════════════════════════════

    1. 发送 Warning 告警到监控群

    3. 继续推进到下一步

═══════════════════════════════════════════════════════════
  🔄 Spawn 重试机制
═══════════════════════════════════════════════════════════

  任何 agent spawn 失败时：
    1️⃣  第一次失败 → 立即重试（相同参数）
    2️⃣  第二次失败 → 等待 10 秒后重试
    3️⃣  第三次仍失败 → 推送告警到监控群 + 通知晨星
                      → 标记该步骤为 BLOCKED

  🚫 绝不因单次 spawn 失败就跳过步骤或 HALT
```
