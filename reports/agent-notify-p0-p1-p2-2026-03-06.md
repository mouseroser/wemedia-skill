# Agent 通知链路 P0 / P1 / P2 执行记录

## P0 止血（已执行）
- 目标：立即避免“任务做了但群里像没发生过”
- 规则：`main` 在每次指派/`spawn` 前补发“开始”，收到结果后补发“完成”或“失败”
- 可靠链路：`main -> 职能群 + 监控群`
- agent 自推：保留，但只算 `best-effort`

## P1 定位（已完成抽样）
### 抽样对象
- `gemini`
- `notebooklm`
- `brainstorming`

### 抽样结论
- 在“单独消息测试” transcript 中，能看到独立发送测试消息的记录
- 在“真实任务” transcript 中，基本看不到对应的 `message` 调用
- 结论：问题不在单条消息权限，而在真实任务链路中的自推执行/编排通知机制不可靠
- 之前“群消息推送 bug 已解决”的方向判断有误，现已纠偏

## P2 决策（当前生效）
- 不再把 agent 自推当成任何可靠通知依据
- agent 群自推继续保留为展示链路，不作为状态同步链路
- 可靠通知职责统一收敛到 `main`
- 后续若要继续修 agent 自推，只能作为增强项，不影响现有可靠通知设计

## 当前执行口径
1. 晨星私聊里，`main` 只做自己的本职工作：接收任务、判断、分发、汇总、补发通知
2. 长任务不在主私聊会话里硬跑
3. 所有关键开始/完成/失败通知，统一由 `main` 发出
4. 单条 `message` 测试成功，今后不得再作为“真实任务推送已修复”的证据
