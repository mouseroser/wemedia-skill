# 记忆质量系统性改进计划

**创建时间**: 2026-03-10 14:26
**目标**: 落地"原话做证据，提炼做索引，推断默认谨慎"原则

## 当前状态

- 总记忆数: 190 条
- 分类: other(80), decision(48), fact(28), preference(12), entity(7), reflection(15)
- 问题: 很多记忆是提炼摘要，缺少原话和出处

## 改进方向

### 1. 记忆质量审计
- 检查最近 50 条记忆
- 识别哪些是"提炼摘要"而非"原话"
- 标记需要补充原话的记忆

### 2. 原话溯源
- 为关键记忆补充原话
- 添加时间戳和对话出处
- 建立"摘要→原话"的追溯链

### 3. 分级存储
- 稳定信息: preference, decision (长期保留)
- 临时信息: other, fact (定期清理)
- 高优先级: 晨星的明确指令、纠正、偏好

### 4. 冲突解决
- 检查是否有冲突的记忆
- 按优先级: 当前对话 > 最近原话 > 历史记忆 > 推断摘要

## 执行步骤

### Step 1: 审计最近 50 条记忆
- 读取最近 50 条记忆
- 分类: 有原话 / 只有摘要 / 有推断
- 生成审计报告

### Step 2: 补充原话
- 优先处理 preference 和 decision 类别
- 从 memory/YYYY-MM-DD.md 找原话
- 更新记忆，添加原话和出处

### Step 3: 清理临时信息
- 识别过时的 other 和 fact
- 删除测试记忆（如刚才的测试记录）
- 保留有价值的技术踩坑

### Step 4: 建立追溯机制
- 为每条重要记忆添加 source 字段
- 格式: "2026-03-10 memory/2026-03-10.md 晨星原话"
- 确保可以追溯到原始对话

## 预期成果

- 所有 preference 和 decision 都有原话
- 每条记忆都有时间和出处
- 临时信息被清理
- 建立了"摘要→原话"的追溯链
