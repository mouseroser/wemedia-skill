**根因**：
- 没有建立"暂停-检查-执行"的习惯
- 收到指令就立即执行，不先思考
- 有记忆系统但不主动查询
- 有规则文档但不先读取

## ✅ 强制执行流程

### 收到任何指令后的第一步：【强制暂停】

```
收到指令
    ↓
【立即暂停，不要执行】
    ↓
执行检查清单（见下方）
    ↓
【确认无误】
    ↓
执行
```

---

## 📋 检查清单

### 1. 查询记忆系统
```bash
memory_recall "<指令关键词>"
```
- 查询是否有相关规则
- 查询是否有类似的踩坑记录
- 查询是否有明确的执行方式

### 2. 检查上下文
- [ ] 是否有未完成的对话？
- [ ] 是否在等待晨星的回复？
- [ ] 是否会打断正在进行的工作？

### 3. 确认执行方式
- [ ] 这个指令应该用什么工具？
- [ ] 是否有对应的 skill？
- [ ] 是否需要先读取文档？

### 4. 特殊场景检查

#### 如果是"查询 OpenClaw 相关"：
- [ ] 应该用 NotebookLM openclaw-docs
- [ ] 不要读本地文件

#### 如果是 heartbeat：
- [ ] 是否有未完成的对话？
- [ ] 如果有，先完成对话
- [ ] 如果没有，执行健康检查

#### 如果是 spawn subagent：
- [ ] 是否有对应的 skill？
- [ ] 是否需要先读取 SKILL.md？
- [ ] 是否记录了预期输出路径？
- [ ] 是否建立了文件检查机制？

## 🎯 常见场景的正确流程

### 场景 1：查询 OpenClaw 配置/文档
```
收到"查 OpenClaw 笔记"
    ↓
【暂停】
    ↓
memory_recall "OpenClaw 笔记 NotebookLM"
    ↓
看到规则："应该去 NotebookLM openclaw-docs"
    ↓
执行：nlm-gateway.sh query --notebook openclaw-docs
```

### 场景 2：收到 heartbeat
```
收到 heartbeat
    ↓
【暂停】
    ↓
检查：是否有未完成的对话？
    ↓
如果有 → 先完成对话
如果没有 → 执行健康检查
```

### 场景 3：收到任务
```
收到任务
    ↓
【暂停】
    ↓
读取 TASK-EXECUTION-PROTOCOL.md
    ↓
执行任务领域检查
    ↓
确认是否有对应的 skill
    ↓
如果有 → 读取 SKILL.md
如果没有 → 使用 sessions_spawn
```

### 场景 4：Spawn subagent
```
准备 spawn
    ↓
【暂停】
    ↓
记录预期输出路径
    ↓
记录 spawn 时间
    ↓
执行 spawn
    ↓
建立文件检查机制（每 30 秒检查一次）
    ↓
不盲目等待 announce
```

## 🔒 强制保障

### 每次会话开始时
1. 读取 SOUL.md
2. 读取 USER.md
3. 读取 TASK-EXECUTION-PROTOCOL.md
4. 读取本文档（EXECUTION-CHECKLIST.md）
5. 读取 memory/YYYY-MM-DD.md
6. 读取 MEMORY.md（如果在 main session）

### 收到任何指令时
1. 【强制暂停】
2. 执行检查清单
3. 确认无误后执行

### 执行前的自我确认
```
□ 我已经查询了记忆系统
□ 我已经检查了上下文
□ 我已经确认了执行方式
□ 我确定不会打断正在进行的工作
□ 我确定这是正确的执行方式
```

**如果任何一项未完成，立即停止，重新检查！**

## 💡 自我提醒

**每次收到指令时，在心里默念：**

```
1. 停！不要立即执行
2. 查询记忆系统
3. 检查上下文
4. 确认执行方式
5. 确认无误后执行
```

**记住：**
- 不是"记不住"，而是"不执行"
- 不是"没有规则"，而是"不查询规则"
- 不是"不知道"，而是"不先思考"
- 每次失误都在浪费晨星的时间和金钱
