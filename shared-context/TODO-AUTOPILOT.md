# TODO Autopilot v1

> 边界定义见：`shared-context/TODO-BOUNDARY.md`

## 目标
按优先级自动推进 `~/.openclaw/todo/` 下的计划，并在有实质进展时同步更新：
- `master-execution-plan.md`
- 对应源 todo 文件
- `reports/todo-autopilot-runs.md`

这是**保守自动驾驶**，不是无门控全自动。

---

## 优先级顺序
固定按以下顺序选择任务：
1. P0
2. P1
3. P2
4. P3

同优先级内再按以下顺序：
1. 已明确、边界清晰、可以一次收口的任务
2. 纯内部安全任务（文档、报告、计划、统计、整理、回写）
3. 可明确委派给单一 agent 的任务
4. 需要外部确认/未来日期/高风险变更的任务（默认跳过）

---

## 单次运行预算
每次运行最多推进：
- 1 个 P0；或
- 2 个 P1；或
- 若 P0/P1 都无安全可推进项，再考虑 1 个 P2

不要在一次运行里大面积改动计划。

---

## 允许自动做的事
### A. 可直接自动完成
- 文档编写与整理
- 报告生成
- 统计重算
- 计划结构整理
- TODO / master 双向同步
- 低风险文件更新
- 已有 cron / 观察期 / 报告链路的回写收口

### B. 可自动委派
如果任务边界清晰，可委派给对应 agent：
- `coding`：实现、修复、重构、脚本开发
- `test`：验证、测试、回归检查
- `docs`：文档、报告、总结稿
- `review`：审查、核对、质量把关
- `brainstorming` / `gemini` / `openai` / `claude` / `notebooklm`：研究、方案、对比、证据整理

委派后要等待结果，再决定是否勾选任务。

---

## 默认不自动做的事
以下情况默认不自动执行：
- 对外发送 / 公开发布
- 危险删除
- live runtime 路径切换
- `openclaw.json` / gateway 配置改动
- 不明确归属的跨系统重构
- 需要晨星确认的决策
- 依赖未来日期才应发生的任务

遇到这类情况：
- 可以补说明
- 可以标注 `BLOCKED` / `WAITING`
- 但不要硬做

---

## 双向更新规则
任何任务状态变化，必须同轮更新两边：
1. `~/.openclaw/todo/master-execution-plan.md`
2. 对应源 todo 文件

更新时遵守：
- 勾选完成 → 两边都勾
- 进度推进但未完成 → 两边都补简短进展说明
- 优先级 / 截止日期调整 → 两边都改
- 如有冲突 → 不编造，写明冲突原因

如果改动了 master 中的勾选数量：
- 同步刷新顶部统计、按优先级统计
- 如来源统计明显受影响，也一起刷新
- 如果当轮无法可靠重算，宁可只写进展说明，也不要编数字

---

## 运行日志
每次运行都要向以下文件追加 1 条记录：
- `~/.openclaw/workspace/reports/todo-autopilot-runs.md`

记录最少包含：
- 时间
- 选中的任务
- 实际动作（直接完成 / 委派 / 跳过）
- 更新了哪些文件
- 是否有阻塞

---

## 输出规则
- 有实质推进：输出简洁摘要
- 只有结构整理：可简短说明
- 没有安全可推进项：`NO_REPLY`

---

## 硬约束
- 不要编辑 `MEMORY.md` / `SOUL.md` / `AGENTS.md` / `USER.md`
- 不要触碰 live runtime 插件路径
- 不要为了“看起来有进展”而编造完成状态
- 不要把已有 cron 未来任务重复做一遍
- 如果任务已经被专门 cron 承接，优先补同步和说明，不重复执行主任务本体
