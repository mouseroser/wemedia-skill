## 1. 总纲

这是一个**共享底盘 + 多业务专用架构并存演化**的多智能体系统。

不是：
- 通用 agent OS
- 单一自媒体系统

而是：
- 在一个共享底盘上，同时承载自媒体、星鉴、星链三类业务架构
- 工程结构服务业务架构，不是反过来

---

## 2. 共享底盘的核心判断

### 系统哲学
- **文件语义固定，数量可增长** — 同一个模型，随着文件系统积累而持续进化
- **业务架构先行，工程架构后随** — 底盘不压平业务架构差异
- **可观察性第一** — 通过文件系统、日志、监控群实现透明协作
- **系统稳定优先** — 任何新功能不能牺牲稳定性和可回滚性

### 执行哲学
- **知道规则就执行** — 不问"要我…吗？"
- **Text > Brain** — 重要结论写进文件
- **踩坑即记** — 防止重复
- **先解再问** — 遇到障碍先解决，卡住了再问

### 通知哲学（B+ 方案）
- 职能群：各 agent 自推（best-effort）
- 监控群：main 只在终态/异常推送
- 晨星 DM：确认门控 / BLOCKED / 最终结果 / 长任务摘要

### 记忆哲学
- Layer 1（文件层）：人类可读、可追溯
- Layer 2（检索层）：日常快速召回
- Layer 3（深理解层）：归档、深研究、跨会话洞察
- 不是三层一起抢活，90% 日常靠 Layer 2

## 3. 三条业务主架构

### 自媒体架构（wemedia pipeline v1.1）
面向持续内容运营的闭环：

信号扫描 → 选题判断 → 宪法边界 → 创作生产 → 配图 → 平台适配 → 发布门控 → 发布执行 → 复盘 → 回流

关键判断：
- Constitution-First 前置链，不跳过宪法直接创作
- 晨星确认门控，未经确认绝不发布
- 发布频率约束（单日 ≤3 篇、间隔 ≥3h）
- 发布禁用 MiniMax
- NotebookLM 深研（M/L 级）+ 衍生内容

已验证能力：
- 小红书全链路（CDP 9223 + 独立 profile）
- NotebookLM infographic 全自动
- 累计 18 篇发布，爆款 Claude HUD 10K（12.9% 互动率）
- 连发限流教训 → ≥3h 间隔规则

### 星鉴架构（stareval pipeline v2.0）
面向研究 / 方案 / 交付：

分级 → 宪法简报 → NotebookLM 深度研究 → 复核 → 仲裁 → 交付

关键判断：
- 轻量宪法 + NotebookLM 主研究双引擎
- 规则驱动 + 资料驱动，避免"资料漂移"
- 首次完整运行已验证通过（2026-03-13）

### 星链架构（starchain pipeline v2.8）
面向项目开发 / 工程实现：

分级 → 宪法 → 方案 → Spec-Kit → coding → review → test → docs → 交付

关键判断：
- Main 直接编排，Review 只审查
- NotebookLM 提前介入打磨层，证据驱动宪法
- Brainstorming 动态模型分配
- Spec-Kit 落地（spec / plan / tasks / research）
- 增强 Epoch 诊断与仲裁
