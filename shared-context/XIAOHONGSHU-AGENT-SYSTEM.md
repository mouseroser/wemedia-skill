## 执行摘要

本方案设计了一套基于 OpenClaw 平台的**小红书 AI 辅助运营系统**，通过 4 个 AI Agent 协作，覆盖内容策划、内容创作、数据分析、互动管理四大核心环节。

### 核心定位
**AI 辅助 + 受控数据读取 + 人工写入执行** — AI Agent 负责"想"和"准备"，人工负责"执行"和"判断"。

### 关键结论
1. **推荐路线**: AI 辅助 + 人工执行模式（非自动化运营）
2. **Agent 架构**: 4 个专职 Agent（雷达 + 工匠 + 互动 + 分析）
3. **MCP 集成**: 采用**方案 C+**（增强型只读集成）— 仅使用 xiaohongshu-mcp 的数据读取功能，禁止写入自动化
4. **实施周期**: 3 阶段 12 周渐进式启用
5. **月运营成本**: ¥300-1000（根据模型选择）
6. **核心价值**: 内容准备效率提升显著，封号风险极低

## 二、xiaohongshu-mcp 集成方案 C+

### 2.1 方案定位

**方案 C+（增强型只读集成）**：
- 在方案 C（纯只读）基础上，将 MCP 只读能力深度嵌入现有 Agent 工作流
- 写入操作全部禁止，保持人工执行
- 使用 MCPorter 配置实现访问控制，而非独立开发中间件

### 2.2 功能分级矩阵

| 功能 | MCP 工具名 | 灯号 | 执行方式 | 日均上限 | 回退方案 |
|------|-----------|------|---------|---------|---------|
| 登录状态检查 | `check_login_status` | 🟢 绿灯 | MCP 自动 | 无限制 | 人工检查 |
| 获取登录二维码 | `get_login_qrcode` | 🟢 绿灯 | MCP + 人工扫码 | 按需 | 人工登录 |
| 删除 Cookies | `delete_cookies` | 🟢 绿灯 | MCP 自动 | 按需 | 手动清除 |
| 搜索内容 | `search_feeds` | 🟢 绿灯 | MCP 自动 | ≤20 | 人工搜索 + 截图 |
| 首页推荐 | `list_feeds` | 🟢 绿灯 | MCP 自动 | ≤10 | 人工浏览 |
| 帖子详情 | `get_feed_detail` | 🟢 绿灯 | MCP 自动 | ≤40 | 人工查看 + 截图 |
| 用户主页 | `user_profile` | 🟡 黄灯 | MCP 自动 | ≤10 | 人工查看 |
| **发布图文** | `publish_content` | 🔴 红灯 | **禁止** | — | 人工发布 |
| **发布视频** | `publish_with_video` | 🔴 红灯 | **禁止** | — | 人工发布 |
| **发表评论** | `post_comment_to_feed` | 🔴 红灯 | **禁止** | — | AI 建议 + 人工回复 |
| **回复评论** | `reply_comment_in_feed` | 🔴 红灯 | **禁止** | — | AI 建议 + 人工回复 |
| **点赞** | `like_feed` | 🔴 红灯 | **禁止** | — | 人工点赞 |
| **收藏** | `favorite_feed` | 🔴 红灯 | **禁止** | — | 人工收藏 |

**日均调用总量**: 约 23-40 次，远低于宪法约束的 200 次/天上限。

### 2.3 MCP Gateway 实现路径

**核心原则**: 使用 MCPorter 配置实现访问控制，预计开发量 2-4 小时（MVP 方式）。

### 2.4 各 Agent 与 MCP 对接

| Agent | 使用 MCP | 对接工具 | 日均调用 |
|-------|---------|---------|---------|
| xhs-radar | ✅ 主用户 | `search_feeds`, `list_feeds` | 15-20 |
| xhs-writer | ❌ 无 | — | 0 |
| xhs-social | ✅ 辅助 | `get_feed_detail` | 5-10 |
| xhs-analyst | ✅ 数据 | `get_feed_detail`, `user_profile` | 5-10 |

## 三、已知失败信号清单

### 3.1 风险类型

| 风险类型 | 描述 | 严重程度 |
|---------|------|---------|
| 账号封禁 | 使用非官方自动化工具被检测后封号 | 致命 |
| 限流处罚 | 笔记被平台减少推荐流量 | 高 |
| 验证码触发 | 登录或操作时要求图形验证 | 中 |
| Cookies 失效 | 登录状态过期需重新扫码 | 中 |
| 实名认证触发 | 新号/未实名账号被要求认证 | 中 |
| IP 限制 | 请求被限制或封禁 | 中 |

### 3.2 降级策略

```
MCP 降级瀑布：
Level 0: MCP 正常 ──────────── 全功能使用
Level 1: MCP 限流 ──────────── 降低调用频率 50%
Level 2: MCP 熔断 ──────────── 停用 MCP，使用备用数据源
Level 3: Cookies 失效 ────────── 停用 MCP，等待人工重新登录
Level 4: 账号异常 ──────────── 停用一切自动化，仅人工操作
```

## 五、宪法约束

- **C1**: 合规第一 — 只读允许，写入禁止
- **C2**: 质量优先于数量 — 三阶段审核
- **C3**: 人工执行门控 — 发布/回复/私信全人工
- **C4**: 数据安全与隐私
- **C5**: 透明度 — 完整审计日志
- **C6**: 可逆性 — MCP 可一键停用
